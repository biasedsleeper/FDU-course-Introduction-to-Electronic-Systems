# 程序结构简述 —— 小车避障绕圈项目

**以下内容由Deepseek V4生成 可能包含错误解释**

## 一、项目概述

本项目运行在树莓派上，使用 Python 编写。小车通过**摄像头**识别彩色目标、**超声波传感器（KS103）**检测前方障碍物距离、**编码器**读取左右轮转速，驱动直流电机完成**避障与绕圈**任务。整个赛道分为 5 个阶段，每阶段根据识别到的颜色（红/黄/蓝）依次导航至目标区域，过程中进行障碍物规避和弧线绕行。

经过先前的所有课程已经完成了此程序的所有前置依赖 此项目需要使用OpenCV、numpy、GPIO等

## 三、main 函数流程

`main()` 的执行分为以下几步：

1. **GPIO 初始化**
   - 设置 BCM 引脚编号模式
   - 配置电机驱动引脚（EA, I2, I1, EB, I4, I3）为输出
   - 配置编码器引脚（LS, RS）为输入，并注册上升沿中断回调
   - 创建两路 PWM 对象（50Hz），分别控制左右电机使能端

2. **启动后台线程**（详见[第五节](#五线程架构)）
   - 线程 1：编码器测速线程 (`getspeed`)
   - 线程 2：超声波测距线程 (`distance_thread`)
   - 线程 3：摄像头视觉线程 (`color_thread`)
   - 休眠 2 秒等待传感器预热

3. **顺序执行 5 个阶段**（`while count == 0`，仅执行一轮）

   | 阶段 | 动作 | 传感器依据 |
   |---|---|---|
   | 阶段 0 | 面向红/黄色目标，直行至距离 < 50cm | 摄像头 + 超声波 |
   | 阶段 1 | 两次 PID 弧线转弯（顺/逆时针各 80°），绕过障碍物 | 编码器（闭环） |
   | 阶段 2 | 搜索并面向黄/蓝色目标，直行至距离 < 45cm | 摄像头 + 超声波 |
   | 阶段 3 | 三次 PID 弧线转弯（顺时针 195°，半径 50cm），进入蓝色区域 | 编码器（闭环） |
   | 阶段 4 | 搜索并面向蓝色目标，直行至距离 < 48cm，最后弧线 + 直行收尾 | 摄像头 + 超声波 + 编码器 |

4. **停止与清理**：捕获 `KeyboardInterrupt` 或自然结束 → 设置停止标志 → 停止 PWM → 释放摄像头 → 清理 GPIO

## 四、关键函数说明

### 4.1 PID 控制器（类 `PID`）

```python
class PID:
    def update(self, feedback_value):
        # error = target - feedback
        # P = Kp * error
        # I += Ki * error (积分累加)
        # D = Kd * (error - prev_error) / dt
        # output = clamp(P + I + D, 0, 100)
```

用于**闭环速度控制**：将编码器实测转速与目标转速比较，输出 PWM 占空比。

### 4.2 运动控制函数

| 函数 | 原理 |
|---|---|
| `go_straight(t)` | 左右轮各一个 PID 控制器，以目标转速（左右略有偏差以补偿机械差异）直行指定秒数 |
| `go_straight_until_dist(target_dist)` | 直行并持续读取全局 `distance`，当距离低于阈值时停车 |
| `turnleft(t)` / `turnright(t)` | 开环 PWM，原地旋转指定秒数 |
| `arc_turn_pid(angle, radius, time)` | **运动学模型**：根据轮距和轮径计算内外轮理想转速差，双 PID 闭环实现弧线轨迹 |
| `face_target()` | **视觉伺服**：读取摄像头画面中目标颜色的 x 坐标 `cx`，与画面中心（160）比较，分段 PID 控制原地转向，使车头对准目标 |
| `search()` | 原地左转并持续检查 `found` 标志，直到摄像头找到目标颜色或超时 |

### 4.3 传感器与视觉函数（`functions.py`）

| 函数 | 原理 |
|---|---|
| `getDistance()` | I2C 向 KS103 发送 `0xb0` 命令，40ms 后读取 2 字节距离值（mm → cm） |
| `init_camera()` | 打开 `/dev/video0`，320×240，7fps，清空 5 帧缓冲后返回 |
| `process_frame(img)` | **完整视觉流水线**：高斯模糊 → BGR 转 HSV → `cv2.inRange` 多组颜色掩膜 → 屏蔽底部 1/3（防地面反光）→ 选最大轮廓 → 计算图像矩得到质心 `(cx, cy)` |
| `show_camera()` | 读取帧 → 调用 `process_frame` → 绘制边界框与质心 → `cv2.imshow` 显示 |

## 五、线程架构

> 这是本项目最核心的设计部分。所有后台任务以**守护线程**形式运行，通过**全局变量**进行无锁通信。

### 5.1 线程总览

```
┌─────────────────────────────────────────────────────────────┐
│                        主线程 (main)                         │
│  执行5阶段顺序逻辑：face_target → go_straight → arc_turn...  │
│  读取全局变量做决策，设置控制标志（RED/BLUE/YELLOW_finished） │
└──────────┬──────────┬──────────┬────────────────────────────┘
           │          │          │
    ┌──────▼──┐ ┌─────▼───┐ ┌───▼──────────┐
    │ 编码器   │ │ 超声波   │ │   摄像头      │
    │ 线程     │ │ 线程     │ │   线程        │
    │ getspeed │ │distance  │ │ color_thread  │
    │ daemon   │ │_thread   │ │   daemon      │
    │          │ │ daemon   │ │               │
    └────┬─────┘ └────┬─────┘ └───┬───────────┘
         │            │            │
         ▼            ▼            ▼
    lspeed,rspeed  distance   cx,cy,current_color
    (转/秒)        (cm)       AREA_amount, found
```

### 5.2 各线程详细说明

#### 线程 1：编码器测速线程 `getspeed()`

- **启动方式**：`threading.Thread(target=getspeed, daemon=True).start()`
- **循环周期**：每 **100ms** 一次
- **工作原理**：
  1. GPIO 上升沿中断回调 `my_callback(channel)` 对左右轮脉冲分别计数
  2. 每 100ms 将累计脉冲数除以 **585**（每转脉冲数），换算为转速（转/秒）
  3. 清零计数器，进入下一周期
- **写入全局变量**：`lspeed`（左轮转速）、`rspeed`（右轮转速）
- **停止条件**：主线程设置 `stop_getspeed = True`

#### 线程 2：超声波测距线程 `distance_thread()`

- **启动方式**：`threading.Thread(target=distance_thread, daemon=True).start()`
- **循环周期**：每 **50ms** 一次
- **工作原理**：调用 `functions.getDistance()` 通过 I2C 读取 KS103 传感器数值
- **写入全局变量**：`distance`（前方障碍物距离，单位 cm）
- **停止条件**：主线程设置 `stop_sensor = True`

#### 线程 3：摄像头视觉线程 `color_thread()`

- **启动方式**：`threading.Thread(target=color_thread, daemon=True).start()`
- **循环周期**：每 **50ms** 一次
- **工作原理**：
  1. 调用 `functions.show_camera()` 抓取并处理一帧图像
  2. 进行 HSV 颜色识别，得到当前检测到的颜色和质心坐标
  3. 综合判断：距离 < 550mm **且** 轮廓面积 > 250 像素 **且** 颜色非 "None" → `found = True`
- **写入全局变量**：`current_color`、`cx`、`cy`、`AREA_amount`、`found`
- **读取全局变量**：`RED_finished`、`BLUE_finished`、`YELLOW_finished`（主线程设置，控制哪些颜色阶段已完成，从而影响 `process_frame` 中激活的颜色掩膜）
- **停止条件**：主线程设置 `stop_sensor = True`

### 5.3 同步机制：无锁全局变量设计

本项目**不使用互斥锁、信号量或队列**，而是利用以下特性保证安全：

1. **写隔离**：每个全局变量只由**唯一的线程**写入，不存在写-写冲突
   - `lspeed/rspeed` → 仅编码器线程写入
   - `distance` → 仅超声波线程写入
   - `cx/cy/current_color/AREA_amount/found` → 仅视觉线程写入
   - `RED/BLUE/YELLOW_finished` → 仅主线程写入

2. **多读安全**：主线程和视觉线程读取其他变量时，由于 CPython 的 **GIL（全局解释器锁）**，单个字节码操作是原子的，Python 简单类型的读写通常不会出现数据损坏

3. **控制标志**：`stop_getspeed`、`stop_sensor` 是简单的布尔标志，主线程设置、工作线程轮询读取，实现安全退出

4. **守护线程**：所有三个工作线程都设置为 `daemon=True`，主线程退出时它们会自动终止，无需手动 join

## 六、避障与绕圈整体思路

### 6.1 核心策略

本项目采用**基于视觉颜色的分段导航 + 距离阈值避障**策略：

1. **颜色地标导航**：赛道关键位置放置红、黄、蓝三色标记。小车通过摄像头识别当前阶段的目标颜色，用 `face_target()` 视觉伺服对准颜色块中心，然后直行接近。

2. **距离阈值避障**：超声波传感器持续测量前方距离。`go_straight_until_dist()` 在距离小于阈值（各阶段分别为 50/45/48 cm）时自动停车，防止碰撞。

3. **弧线绕行**：在阶段 1 和阶段 3 中，使用 `arc_turn_pid()` 执行精确的弧线轨迹（基于运动学模型的差速驱动），绕开或绕过障碍物/赛道边界。

4. **丢失搜索**：当目标颜色暂时不可见时，`search()` 让小车原地旋转并持续检测，直到重新捕获目标。

### 6.2 控制环路层次

```
┌─────────────────────────────────────────────────┐
│                   高层决策层                       │
│   main() 阶段机：决定"现在该做什么"                │
│   依据：阶段计数 + 颜色完成标志 + distance + found  │
└────────────────────┬────────────────────────────┘
                     │ 调用
┌────────────────────▼────────────────────────────┐
│                   中层运动原语                     │
│   face_target / go_straight / arc_turn / search  │
│   依据：cx（视觉）+ distance（超声波）+ speed（编码器）│

                     │ 调用

│                   底层 PID 控制                    │
│   双 PID 闭环：目标转速 vs 编码器实测转速 → PWM     │
│   分段 PID：cx 偏差 → 转向 PWM                     │
└─────────────────────────────────────────────────┘
```

## 七、总结

本项目的整体架构可以概括为：

> **三线程后台感知 + 主线程顺序决策 + PID 闭环执行**

- **感知层**：三个守护线程分别以不同频率独立采集编码器转速、超声波距离、摄像头图像，写入全局变量
- **决策层**：主线程按预设 5 阶段顺序调用运动原语，根据全局变量判断何时切换动作
- **执行层**：双 PID 控制器（速度环 + 视觉伺服）将高层指令转化为精确的 PWM 输出，驱动电机完成运动

该设计的优势在于**简单直接**：无锁共享内存避免了复杂的同步开销，守护线程自动管理生命周期，PID 闭环保证了运动的准确性。局限性在于阶段逻辑硬编码，不具备动态路径规划能力，适用于固定赛道场景。
